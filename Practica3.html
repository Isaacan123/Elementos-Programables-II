Practica 3:

Velocidad del microcontrolador
Resumen

Resumen

El objetivo de esta práctica es comparar el tiempo de respuesta en la comunicación serie entre diferentes microcontroladores: el ATtiny86, el ESP32-C6 y el Arduino Uno. La metodología se centra en enviar un dato (letra "a") desde un computador hacia el microcontrolador mediante comunicación serie USB, y medir el tiempo que tarda en recibir una respuesta, que también será la letra "a". Para esto se utilizaron dos plataformas: Arduino IDE para programar los microcontroladores y Visual Studio con Python para el monitoreo de la comunicación y la medición de los tiempos de respuesta.

Introducción

Introducción

La comunicación serie es uno de los métodos más utilizados para la transmisión de datos entre un microcontrolador y un computador, especialmente cuando se requiere simplicidad y eficacia en la transmisión. Este tipo de comunicación se realiza a través de puertos de tipo USB-serial, y la velocidad de transmisión de datos está determinada principalmente por la velocidad de baudios y la eficiencia de la implementación en el software y hardware.

En esta práctica se compararon tres microcontroladores de características diferentes:

  • Attiny: de pequeña escala, enfocado en aplicaciones de bajo consumo y con recursos limitados.
  • Arduino Uno: Microcontrolador de uso común, basado en el chip ATmega328P, con soporte amplio y comunidad activa.
  • ESP32-C6: Microcontrolador más avanzado con capacidades de conectividad Wi-Fi y Bluetooth.

Objetivo

Objetivo

El objetivo de esta práctica es evaluar el rendimiento de diferentes microcontroladores en la comunicación serie, midiendo y comparando los tiempos de respuesta al enviar y recibir datos (la letra "a") mediante comunicación USB-serial. Esto permitirá identificar cuál microcontrolador es más eficiente en términos de latencia, considerando las características específicas de cada uno: el ESP32-C6, el Arduino Uno y el ATtiny86.

Procedimiento

Procedimiento

1.-Se programaron los tres microcontroladores con el mismo código básico que consiste en:

  • Configurar el puerto serie a una
    velocidad de 115200 baudios.
  • Leer datos recibidos por el puerto serie.
  • Responder al recibir la letra "a"
    enviando de nuevo la misma letra.
2.-En la computadora, se utilizó un script en Python (con la librería pySerial) para realizar las pruebas de comunicación:
  • Enviar la letra "a" al microcontrolador.
  • Medir el tiempo transcurrido desde que se envió la letra hasta que se recibe la respuesta.
  • Repetir el proceso múltiples veces
    (100 veces por cada microcontrolador).
3.-Los tiempos de respuesta fueron recopilados y promediados al final de cada serie de pruebas.

ESP32-C6

Codigo en Visual studio code

import serial
import time

# Configurar el puerto serie
SERIAL_PORT = "COM4"  
BAUD_RATE = 115200
NUM_TESTS = 100  # Número de veces que se enviará "a"

# Abrir conexión serie
ser = serial.Serial(SERIAL_PORT, BAUD_RATE, timeout=1)
time.sleep(2)  # Esperar a que el puerto se estabilice

# Lista para almacenar tiempos de respuesta
response_times = []

try:
for i in range(NUM_TESTS):
    start_time = time.time()  # Tomar tiempo antes de enviar
    ser.write(b'a')  # Enviar la letra "a"
    
    response = ser.read(1)  # Leer respuesta
    
    if response == b'a':  # Verificar si la respuesta es correcta
        round_trip_time = (time.time() - start_time) * 1000  # Convertir a ms
        response_times.append(round_trip_time)
        print(f"Prueba {i+1}: {round_trip_time:.3f} ms")
    else:
        print(f"Prueba {i+1}: Error, no se recibió respuesta")

    time.sleep(0.01)  # Pequeña pausa para estabilidad

# Calcular el promedio de tiempos
if response_times:
    avg_time = sum(response_times) / len(response_times)
    print(f"\nTiempo promedio de respuesta: {avg_time:.3f} ms")
else:
    print("\nNo se obtuvieron respuestas válidas.")

except KeyboardInterrupt:
print("\nInterrupción manual. Cerrando conexión...")

finally:
ser.close()

Resultados

Resultados

Para cada uno de los microcontroladores (ESP32-C6, Arduino Uno y ATtiny86), se obtuvieron tiempos de respuesta en milisegundos. Los resultados exactos pueden variar dependiendo de varios factores como el rendimiento del puerto USB y la eficiencia de los microcontroladores.

Estos resultados indican que, aunque todos los microcontroladores lograron realizar la tarea, el ESP32-C6 mostró la menor latencia, lo que sugiere que este microcontrolador tiene un mejor rendimiento en términos de comunicación serie con respecto a los otros dos, además es el único que puede soportar más de 115200 baudios. El Arduino Uno mostró un rendimiento intermedio, mientras que el ATtiny86 con 9600 baudios, debido a sus limitaciones de hardware, mostró el mayor tiempo de respuesta.

Conclusion

Conclusion

  1. El ESP32-C6 es el microcontrolador con mejor rendimiento en términos de velocidad de respuesta en comunicaciones serie por sus capacidades de procesamiento superiores.
  2. El Arduino Uno, aunque es un microcontrolador más accesible, tiene un tiempo de respuesta ligeramente superior, lo que puede ser una limitación en aplicaciones de alta velocidad.
  3. El ATtiny86 presentó los tiempos de respuesta más altos por su arquitectura más simple y menos optimizada para tareas de comunicación rápidas.

    4. Para aplicaciones que requieran una comunicación rápida y eficiente, el ESP32-C6 es la opción recomendada, especialmente si se busca aprovechar sus capacidades adicionales como Wi-Fi y Bluetooth. Sin embargo, para proyectos con recursos limitados y necesidades de comunicación más simples, el Arduino Uno o el ATtiny86 pueden ser opciones viables dependiendo del contexto.
