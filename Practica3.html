Practica 3:

Velocidad del microcontrolador
Resumen

Resumen

Esta práctica evalúa la velocidad de procesamiento de distintos microcontroladores (ATtiny, ATmega y ESP32) midiendo el tiempo que tardan en recibir y ejecutar la instrucción de imprimir "Hola, mundo" en el monitor serial. Se tomarán 100 mediciones por cada MCU para obtener datos representativos. Los resultados se organizarán en tablas y gráficos para comparar su eficiencia y analizar las diferencias de rendimiento.

Introducción

Introducción

Los microcontroladores son dispositivos clave en el desarrollo de sistemas embebidos y electrónica digital. Evaluar su rendimiento es esencial para determinar cuál es más eficiente en diferentes aplicaciones. En esta práctica, se medirá el tiempo que tarda cada MCU en procesar una instrucción simple, lo que permitirá comparar su velocidad de ejecución.

Análisis de Velocidad en Microcontroladores

El propósito de esta práctica es examinar y comparar la rapidez con la que distintos microcontroladores (MCUs) procesan una instrucción específica.

Attiny
 		ATTMega ESP32

Objetivo

Objetivo

Determinar el tiempo que tarda cada MCU en recibir y ejecutar una instrucción simple, específicamente encender un LED, midiendo así su eficiencia en términos de procesamiento.

Procedimiento

Procedimiento

Se realizará la carga de un código en cada microcontrolador, el cual activará un LED. Se registrará el tiempo desde el inicio de la carga del código hasta la ejecución de la instrucción. Se tomarán 100 mediciones por cada MCU para obtener datos representativos.


Visual studio code & Phyton

El desarrollo de software para microcontroladores (MCU) se inicia configurando Visual Studio Code con Python. Luego, se escribe el código del programa, y finalmente, este código se transfiere al MCU, asegurando su correcta ejecución.

Análisis y Representación de Datos




Los resultados serán organizados en una tabla y representados gráficamente para visualizar y comparar la eficiencia de cada MCU en términos de velocidad de carga y ejecución. Se analizarán posibles variaciones y se discutirán las razones detrás de las diferencias observadas.

Conclusion

Conclusion

El ATtiny y el ATmega simulan multitarea mediante técnicas como interrupciones y multiplexación, mientras que el ESP32 soporta multitarea real gracias a su doble núcleo y FreeRTOS, ofreciendo una mayor eficiencia en la gestión de tareas concurrentes.
